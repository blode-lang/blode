02. Logic Flow

The act Syntax

Forget complex class hierarchies. Blode uses Actions. It defines exactly what should happen and when.

  • Explicit entry points with act main()
  • Property-based system calls
  • Synchronous timing control
example.bld
#contain <sysio>
act main() {
  sys.clear();
  sys.output = "System active.";
  sys.wait(1.5);
  sys.input = "Username:";
}

Why Blode?

Direct Action

Logic is encapsulated in 'actions'. No unnecessary boilerplate, just direct entry points for your system scripts.

System Oriented

The sys object provides native access to terminal controls, timing, and I/O out of the box.

Built-in Simulation

Perfect for creating realistic CLI mockups and automated installation simulations with millisecond precision.

Human Readable.
Machine Precise.

Blode removes the friction between an idea and a working terminal script. By utilizing the #contain directive, you get instant access to powerful system primitives.

Designed for solo developers who need to move fast without fighting complex syntax rules.

demo_program.bld
#contain <sysio>

act main() {
  sys.clear();
  sys.output = "Deploying Blode...";
  sys.wait(2);
  sys.output = "Success!";
  sys.input = "Press Enter to exit.";
}
